## 🎯 部署步骤

### 步骤1：获取微信公众号配置信息

#### 1.1 登录微信公众平台
- 访问：https://mp.weixin.qq.com/
- 使用管理员微信扫码登录

#### 1.2 获取AppID和AppSecret
```
路径：设置与开发 → 基本配置 → 开发者ID(AppID)
```
记录以下信息：
- **AppID**（开发者ID）：类似 `wx1234567890abcdef`
- **AppSecret**（开发者密码）：需要点击"重置"生成，只显示一次，请妥善保存

#### 1.3 检查公众号类型
- **服务号（已认证）**：✅ 可以使用网页授权
- **订阅号**：❌ 无法使用网页授权（需要升级为服务号）
- **未认证服务号**：❌ 需要先完成认证

---

### 步骤2：配置授权回调域名

#### 2.1 在微信公众号后台配置
```
路径：设置与开发 → 接口权限 → 网页服务 → 网页授权 → 网页授权域名
```

#### 2.2 填写你的域名
假设你的域名是 `example.com`，填写：
```
example.com
```
**注意事项**：
- ✅ 只填写域名，不要带 `http://` 或 `https://`
- ✅ 不要带端口号
- ✅ 支持子域名，如 `api.example.com`
- ❌ 不能使用 IP 地址
- ❌ 不能使用 localhost

#### 2.3 下载验证文件
- 微信会提供一个验证文件 `MP_verify_xxx.txt`
- 将这个文件放到你网站根目录
- 确保可以通过 `http://example.com/MP_verify_xxx.txt` 访问

---

### 步骤3：修改服务器配置

#### 3.1 修改 `server.js` 配置
将以下配置替换为你的真实信息：

```javascript
const WECHAT_CONFIG = {
    appId: 'wx1234567890abcdef',           // 替换为你的AppID
    appSecret: 'your_real_app_secret',     // 替换为你的AppSecret
    redirectUri: 'https://example.com/wechat/callback'  // 替换为你的回调地址
};
```

**重要提示**：
- `redirectUri` 必须使用 HTTPS（生产环境）
- 域名必须与微信后台配置的一致
- 回调路径保持为 `/wechat/callback`

#### 3.2 配置生产环境端口
在 `server.js` 中添加环境变量支持：

```javascript
const PORT = process.env.PORT || 3001;
```

### 步骤4：修改客户端API地址

#### 4.1 创建环境配置文件
创建 `js/config.js`：

```javascript
const API_CONFIG = {
    // 生产环境
    production: {
        apiBaseUrl: 'https://example.com',  // 替换为你的域名
        apiPort: ''  // 生产环境不需要端口号（使用443）
    },
    // 开发环境
    development: {
        apiBaseUrl: 'http://localhost',
        apiPort: ':3001'
    }
};

// 自动检测环境
const ENV = window.location.protocol === 'file:' ? 'development' : 'production';
const API_BASE = API_CONFIG[ENV].apiBaseUrl + API_CONFIG[ENV].apiPort;

window.API_BASE = API_BASE;
```

#### 4.2 修改 `profile-module.js`
将所有硬编码的 `http://localhost:3001` 替换为 `window.API_BASE`：

```javascript
// 原代码
const response = await fetch('http://localhost:3001/api/wechat/qrcode');

// 修改为
const response = await fetch(`${window.API_BASE}/api/wechat/qrcode`);
```

需要修改的位置：
- 第473行：生成二维码
- 第603行：检查登录状态
- 第668行：检查关注状态

### 步骤5：准备服务器部署

#### 5.1 服务器环境准备
登录你的腾讯云服务器，确保安装：

```bash
# 安装 Node.js（如果未安装）
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 PM2（进程管理器）
sudo npm install -g pm2

# 安装 Nginx（反向代理）
sudo apt-get install nginx
```

#### 5.2 创建部署目录
```bash
mkdir -p /www/jianying-server
cd /www/jianying-server
```

#### 5.3 上传文件
将以下文件上传到服务器：
- `server.js`
- `package.json`
- `.env`（配置文件，见下方）

#### 5.4 创建 `.env` 配置文件
在服务器上创建 `.env` 文件：

```bash
# 微信配置
WECHAT_APP_ID=wx1234567890abcdef
WECHAT_APP_SECRET=your_real_app_secret
WECHAT_REDIRECT_URI=https://example.com/wechat/callback

# 服务器配置
PORT=3001
NODE_ENV=production
```

#### 5.5 安装依赖
```bash
cd /www/jianying-server
npm install
```

### 步骤6：配置 Nginx 反向代理

#### 6.1 创建 Nginx 配置文件
```bash
sudo nano /etc/nginx/sites-available/jianying-api
```

#### 6.2 添加配置内容
```nginx
server {
    listen 80;
    server_name example.com;  # 替换为你的域名
    
    # 自动重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name example.com;  # 替换为你的域名
    
    # SSL 证书配置（使用腾讯云证书）
    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;
    
    # 微信验证文件
    location ~ ^/MP_verify_.*\.txt$ {
        root /www/website;  # 你的网站根目录
    }
    
    # API 请求转发到 Node.js 服务
    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    # 微信回调路径
    location /wechat/ {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

#### 6.3 启用配置
```bash
sudo ln -s /etc/nginx/sites-available/jianying-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 步骤7：配置 SSL 证书（HTTPS）

#### 7.1 使用腾讯云免费证书
```
腾讯云控制台 → SSL证书 → 申请免费证书
```

#### 7.2 下载证书
- 下载 Nginx 格式的证书
- 包含两个文件：`.crt` 和 `.key`

#### 7.3 上传证书到服务器
```bash
sudo mkdir -p /etc/nginx/ssl
sudo cp example.com.crt /etc/nginx/ssl/
sudo cp example.com.key /etc/nginx/ssl/
sudo chmod 600 /etc/nginx/ssl/*
```

## 🚨 常见问题排查

### 问题1：二维码生成失败
**可能原因**：
- 网络连接问题
- 服务器未启动
- API 地址配置错误

**解决方法**：
```bash
# 检查服务器是否运行
pm2 list

# 检查端口是否监听
netstat -tlnp | grep 3001

# 测试 API 接口
curl http://localhost:3001/api/wechat/qrcode
```

### 问题2：扫码后无反应
**可能原因**：
- 回调域名配置错误
- Nginx 配置错误
- SSL 证书问题

**解决方法**：
- 检查微信公众号后台的授权回调域名
- 确保 HTTPS 可以正常访问
- 查看 Nginx 和 PM2 日志

### 问题3：提示"redirect_uri参数错误"
**原因**：回调地址与微信后台配置不一致

**解决方法**：
1. 检查 `server.js` 中的 `redirectUri`
2. 确保与微信后台"网页授权域名"一致
3. 重启服务：`pm2 restart jianying-api`

### 问题4：获取用户信息失败
**可能原因**：
- AppSecret 错误
- 公众号类型不支持
- 未获得用户授权

**解决方法**：
- 验证 AppID 和 AppSecret 是否正确
- 确认公众号是已认证的服务号
- 检查微信授权流程是否完整

## 🔐 安全建议

1. **不要将 AppSecret 提交到 Git**
   - 使用 `.env` 文件存储敏感信息
   - 将 `.env` 添加到 `.gitignore`

2. **使用环境变量**
   ```javascript
   require('dotenv').config();
   const appId = process.env.WECHAT_APP_ID;
   ```

3. **限制 API 访问频率**
   - 使用 express-rate-limit 中间件
   - 防止恶意刷接口

4. **定期更新依赖**
   ```bash
   npm audit
   npm update
   ```
