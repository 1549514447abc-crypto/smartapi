# 剪映草稿客户端开发项目需求总结

## 项目概述

### 核心目标
开发一个桌面客户端应用，将基于Coze插件生成的剪映草稿JSON自动转换并安装到本地剪映软件中，实现从云端草稿到本地可编辑项目的无缝转换。

### 业务流程
```
用户在Coze完成视频制作 → 插件生成draft_content.json → 上传OSS
↓
插件返回OSS下载链接 → 用户复制链接到客户端
↓
客户端下载JSON → 提取素材URL → 下载素材文件 → 重命名文件
↓
生成完整剪映项目结构 → 写入本地剪映目录 → 用户可在剪映中打开编辑
```

## 关键技术实现

### 🎯 媒体文件路径链接系统 (核心技术)

#### ⚠️ 关键原理 - 切勿修改
经过深入研究和测试验证，剪映的媒体文件路径链接机制如下：

**正确的路径格式**：
```javascript
// ✅ 正确：直接使用绝对路径
"path": "G:\\JianyingPro Drafts\\项目ID\\资源文件夹ID\\文件UUID.mp3"

// ❌ 错误：使用占位符格式
"path": "##_draftpath_placeholder_resourceFolderId_##\\resourceFolderId\\filename"
```

#### 技术依据
基于开源项目 `pyJianYingDraft` 的成熟实现：
- 直接使用 `os.path.abspath(path)` 生成的绝对路径
- 不使用任何占位符或特殊格式
- 与剪映官方格式完全兼容

#### 实现代码 (js/download-core.js)
```javascript
replacePlaceholders(draftData, resourceFolderId, projectPath) {
    // 构建资源文件夹的绝对路径
    const path = window.require('path');
    const resourcePath = path.join(projectPath, resourceFolderId);
    
    // 直接使用绝对路径替换HTTP链接
    material.path = path.join(resourcePath, fileName);
}
```

#### 🚨 重要警告
- **绝对不要**使用占位符格式，这会导致"媒体丢失"
- **绝对不要**使用相对路径，剪映无法识别
- **必须使用**完整的Windows/Mac绝对路径格式

### 🔧 HTTP重定向处理机制

#### 问题背景
Coze图片链接（如 `https://s.coze.cn/t/xxx/`）会返回302重定向，需要正确处理。

#### 解决方案
```javascript
// 检测重定向状态码
if (response.statusCode === 301 || response.statusCode === 302 || 
    response.statusCode === 307 || response.statusCode === 308) {
    const redirectUrl = response.headers.location;
    // 抛出特殊错误，由重试机制处理
    reject(new Error(`REDIRECT:${redirectUrl}`));
}

// 重试循环中处理重定向
if (error.message.startsWith('REDIRECT:')) {
    const redirectUrl = error.message.substring(9);
    url = redirectUrl; // 更新URL
    attempt--; // 重定向不计入重试次数
    continue;
}
```

## 用户认证系统设计

### 🔐 登录系统架构

#### 微信扫码登录 + 公众号关注流程
```
用户点击扫码登录 → 客户端请求二维码 → 显示二维码
↓
用户微信扫码 → 关注公众号 → 微信推送关注事件
↓
后端验证关注 → 生成JWT Token → 客户端获取Token → 登录成功
```

#### 手机号登录流程
```
用户输入手机号 → 发送验证码 → 用户输入验证码
↓
后端验证 → 生成JWT Token → 客户端存储Token → 登录成功
```

### 🏗️ 后端服务需求

#### 必需的API接口
```
POST /api/auth/wechat/qrcode     # 生成登录二维码

POST /api/auth/sms/send          # 发送短信验证码
POST /api/auth/sms/verify        # 验证手机号登录
POST /api/auth/refresh           # 刷新Token
GET  /api/user/profile           # 获取用户信息
```
