### 第四步：部署服务

#### 方式A：自动部署（推荐）

修改 `deploy.sh` 配置：
```bash
SERVER_USER="root"                    # 你的服务器用户名
SERVER_HOST="123.456.789.123"         # 你的服务器IP  
SERVER_PATH="/www/jianying-server"
```
- [ ] 已修改部署脚本配置

执行部署：
```bash
# Mac/Linux
chmod +x deploy.sh
./deploy.sh

# Windows (使用 Git Bash)
bash deploy.sh
```
- [ ] 部署脚本执行成功
- [ ] 服务已启动

#### 方式B：手动部署

如果自动部署失败，使用手动方式：

1. 上传文件到服务器：
```bash
# 本地执行
scp server.js package.json .env ecosystem.config.js root@服务器IP:/www/jianying-server/
```
- [ ] 文件已上传

2. 在服务器上安装依赖：
```bash
cd /www/jianying-server
npm install --production
```
- [ ] 依赖已安装

3. 启动服务：
```bash
mkdir -p logs
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```
- [ ] PM2 服务已启动
- [ ] 已设置开机自启

### 第五步：配置 Nginx

#### 5.1 上传 SSL 证书
```bash
# 在服务器上创建目录
mkdir -p /etc/nginx/ssl

# 从本地上传证书（在本地执行）
scp your-domain.crt your-domain.key root@服务器IP:/etc/nginx/ssl/
```
- [ ] 证书已上传到服务器
- [ ] 证书权限已设置：`chmod 600 /etc/nginx/ssl/*`

#### 5.2 配置 Nginx
```bash
# 在服务器上执行
nano /etc/nginx/sites-available/jianying-api
```

复制 `nginx.conf.example` 的内容，修改：
- [ ] 所有域名已改为真实域名

- [ ] 网站根目录路径已修改

#### 5.3 启用配置
```bash
ln -s /etc/nginx/sites-available/jianying-api /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```
- [ ] Nginx 配置测试通过

## 🔧 故障排查

如果测试失败，按以下顺序检查：

### 检查项 1：服务器服务
```bash
pm2 status                    # 查看服务状态
pm2 logs jianying-api         # 查看错误日志
netstat -tlnp | grep 3001     # 检查端口监听
```

### 检查项 2：Nginx 配置
```bash
nginx -t                      # 测试配置
tail -f /var/log/nginx/error.log  # 查看错误日志
```

### 检查项 3：客户端配置
- 打开浏览器开发者工具（F12）
- 查看 Console 是否有错误
- 查看 Network 标签，检查 API 请求状态

### 检查项 4：微信配置
- 确认 AppID 和 AppSecret 正确
- 确认回调域名配置正确
- 确认验证文件可访问
