# 微信扫码登录配置说明

## 功能概述

已成功实现微信扫码登录功能，用户必须通过微信扫码才能登录，删除了手机号登录选项。

## 实现的功能

1. ✅ **删除手机号登录** - 只保留微信扫码登录
2. ✅ **真实二维码生成** - 使用微信开放平台API生成登录二维码
3. ✅ **实时状态轮询** - 实时检测扫码和登录状态
4. ✅ **公众号关注检测** - 登录后检查用户是否关注公众号
5. ✅ **现代化UI** - 优化的登录界面和用户体验

## 配置步骤

### 1. 安装依赖

```bash
npm install
```

### 2. 配置微信公众号

在 `server.js` 文件中修改以下配置：

```javascript
const WECHAT_CONFIG = {
    appId: 'your_wechat_app_id',           // 替换为你的微信开放平台AppID
    appSecret: 'your_wechat_app_secret',   // 替换为你的微信开放平台AppSecret
    redirectUri: 'http://localhost:3001/wechat/callback'  // 回调地址
};
```

### 3. 微信公众号后台配置

1. 登录微信公众平台 (mp.weixin.qq.com)
2. 进入"开发" -> "接口权限" -> "网页服务" -> "网页授权获取用户基本信息"
3. 设置授权回调域名：`localhost:3001`

### 4. 启动服务

```bash
# 启动后端服务
npm run server

# 启动前端应用（新终端窗口）
npm run dev

# 或者同时启动前后端
npm run dev:full
```

## 登录流程

1. 用户点击"微信扫码登录"按钮
2. 系统生成微信登录二维码
3. 用户使用微信扫描二维码
4. 用户在微信中确认登录
5. 系统检查用户是否关注公众号
6. 登录成功，更新用户信息

## API接口

### 生成二维码
```
GET /api/wechat/qrcode
```

### 检查登录状态
```
GET /api/wechat/status/:sessionId
```

### 检查关注状态
```
GET /api/wechat/check-follow/:openid
```

### 微信回调
```
GET /wechat/callback
```

## 技术实现

- **后端**: Node.js + Express
- **前端**: 原生JavaScript + 轮询机制
- **二维码**: qrcode库生成
- **微信API**: 微信开放平台OAuth2.0
- **状态管理**: 内存存储 + 定时清理

## 注意事项

1. 确保微信公众号已认证
2. 回调域名必须与配置一致
3. 生产环境需要HTTPS
4. 定期清理过期的登录会话

## 测试

1. 启动服务后访问个人中心
2. 点击"微信扫码登录"
3. 使用微信扫描生成的二维码
4. 确认登录后检查是否成功

## 故障排除

- **二维码生成失败**: 检查微信配置是否正确
- **扫码后无响应**: 检查回调域名配置
- **关注检测失败**: 检查公众号权限设置
- **网络错误**: 确保后端服务正常运行
