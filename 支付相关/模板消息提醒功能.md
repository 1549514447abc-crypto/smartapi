# 微信模板消息提醒功能

## 一、功能概述

通过微信公众号模板消息，向用户发送业务通知提醒，提升用户体验。

### 主要应用场景

| 场景 | 触发时机 | 紧急程度 |
|------|----------|----------|
| 提现确认提醒 | 转账发起后立即发送 | 高 |
| 提现即将过期提醒 | 转账发起后20小时 | 高 |
| 提现成功通知 | 用户确认收款后 | 中 |
| 提现失败/过期通知 | 转账失败或过期时 | 中 |
| 订单支付成功 | 用户支付完成后 | 中 |
| 佣金到账通知 | 推荐用户消费产生佣金时 | 低 |

## 三、技术实现方案

### 3.1 后端服务

创建 `WechatTemplateService.ts`：

```typescript
// backend/src/services/WechatTemplateService.ts

interface TemplateMessage {
  touser: string;        // 用户openid
  template_id: string;   // 模板ID
  url?: string;          // 点击跳转链接
  data: Record<string, { value: string; color?: string }>;
}

class WechatTemplateService {
  // 发送模板消息
  async send(message: TemplateMessage): Promise<boolean>;

  // 发送提现确认提醒
  async sendWithdrawalReminder(openid: string, amount: number, expireTime: Date): Promise<boolean>;

  // 发送提现成功通知
  async sendWithdrawalSuccess(openid: string, amount: number): Promise<boolean>;

  // 发送支付成功通知
  async sendPaymentSuccess(openid: string, orderNo: string, amount: number, productName: string): Promise<boolean>;

  // 发送佣金到账通知
  async sendCommissionNotice(openid: string, amount: number, source: string): Promise<boolean>;
}
```

### 3.2 定时任务

创建定时任务检查即将过期的转账：

```typescript
// 每小时执行一次
// 检查距离过期还有4小时的转账，发送提醒

cron.schedule('0 * * * *', async () => {
  // 查找状态为 WAIT_USER_CONFIRM 且创建时间超过20小时的转账
  const pendingTransfers = await WithdrawalTransfer.findAll({
    where: {
      batch_status: 'WAIT_USER_CONFIRM',
      created_at: {
        [Op.lt]: new Date(Date.now() - 20 * 60 * 60 * 1000), // 20小时前
        [Op.gt]: new Date(Date.now() - 24 * 60 * 60 * 1000)  // 24小时内
      },
      reminder_sent: false  // 未发送过提醒
    }
  });

  for (const transfer of pendingTransfers) {
    await wechatTemplateService.sendWithdrawalReminder(...);
    await transfer.update({ reminder_sent: true });
  }
});
```

### 3.3 数据库变更

`withdrawal_transfers` 表添加字段：

```sql
ALTER TABLE withdrawal_transfers
ADD COLUMN reminder_sent TINYINT(1) DEFAULT 0 COMMENT '是否已发送过期提醒';
```

## 五、发送时机

### 5.1 提现相关

```
用户申请提现
    ↓
管理员审批通过，发起微信转账
    ↓
转账状态为 WAIT_USER_CONFIRM
    ↓
【立即发送】提现确认提醒
    ↓
    ├── 用户20小时内未确认 → 【发送】即将过期提醒
    │
    ├── 用户确认收款 → 【发送】提现成功通知
    │
    └── 24小时过期 → 【发送】提现过期通知 + 自动退款
```

### 5.2 订单相关

```
用户支付成功 → 【发送】支付成功通知
```

### 5.3 佣金相关

```
推荐用户消费，佣金到账 → 【发送】佣金到账通知（可选，避免过度打扰）
```

## 七、注意事项

1. **频率限制**：同一用户每天接收模板消息有上限，避免重复发送
2. **内容规范**：严格按照模板要求填写，不得添加营销内容
3. **用户体验**：只发送必要通知，避免骚扰用户
4. **错误处理**：发送失败需要记录日志，便于排查
5. **关注状态**：只有关注公众号的用户才能收到模板消息
