## 系统架构

### 技术栈
- **前端用户端**: React + TypeScript + Vite + Ant Design
- **前端管理端**: React + TypeScript + Vite + Ant Design
- **后端**: Node.js + Express + TypeScript + Sequelize
- **数据库**: MySQL (端口: 13306)
- **进程管理**: PM2
- **Web服务器**: Nginx

### 项目结构
```
smartapi/
├── frontend/           # 用户前端
├── admin/             # 管理后台前端
├── backend/           # 后端服务
```

## 佣金系统

### 佣金计算逻辑

#### 优先级规则
1. **用户自定义比例** > **用户分类默认比例** > **全局比例**
2. 全局设置 = 普通用户分类设置

#### 佣金比例来源
```typescript
// 获取用户实际佣金比例的优先级
async function getCommissionRateForUser(user, sourceType) {
  // 1. 优先使用用户自定义比例
  if (sourceType === 'course' && user.custom_commission_rate_course !== null) {
    return user.custom_commission_rate_course / 100;
  }
  if (sourceType === 'membership' && user.custom_commission_rate_membership !== null) {
    return user.custom_commission_rate_membership / 100;
  }

  // 2. 使用用户分类的默认比例
  if (user.user_category_id) {
    const category = await UserCategory.findByPk(user.user_category_id);
    if (category) {
      if (sourceType === 'course') return category.default_course_rate / 100;
      if (sourceType === 'membership') return category.default_membership_rate / 100;
    }
  }

  // 3. 使用全局配置
  const globalKey = sourceType === 'course'
    ? ConfigKey.COMMISSION_RATE_COURSE
    : ConfigKey.COMMISSION_RATE_MEMBERSHIP;
  return await SystemConfig.getNumberConfig(globalKey) / 100;
}
```

### 佣金结算规则
- **结算周期**: 15天
- **待结算状态**: 佣金产生后进入 `pending_commission_balance`
- **可提现状态**: 15天后自动转入 `commission_balance`
- **结算时间**: 每天凌晨执行自动结算任务

### 佣金来源类型
1. **course**: 推荐用户购买课程
2. **membership**: 推荐用户购买会员
3. **points**: 推荐用户充值（通用比例）

### 管理功能
**管理后台 > 分佣设置**
- 查看所有用户分类及其默认佣金比例
- 创建/编辑/删除用户分类
- 为单个用户设置自定义佣金比例（在用户管理页面）

## 提现系统

### 提现流程

```
用户申请提现
    ↓
检查余额、限额、微信绑定
    ↓
扣除commission_balance（冻结金额）
    ↓
创建withdrawal_request（状态：pending）
    ↓
管理员审核
    ↓
批准 → approved    拒绝 → rejected（退回余额）
    ↓
自动拆单（每笔最多200元）
    ↓
创建withdrawal_transfers记录
    ↓
调用微信企业付款API
    ↓
成功 → completed   失败 → failed（退回余额）
```

### 提现限制配置

| 配置项 | 配置键 | 默认值 | 说明 |
|--------|--------|--------|------|
| 最低提现金额 | `min_withdrawal_amount` | 10元 | 单次提现不能低于此金额 |
| 每日提现上限 | `max_daily_withdrawal` | 2000元 | 每个用户每天最多提现金额 |
| 单笔转账上限 | `max_single_transfer` | 200元 | 微信单笔转账限额，超过自动拆单 |

### 提现状态说明

| 状态 | 说明 |
|------|------|
| `pending` | 待审核 |
| `approved` | 已批准，等待转账 |
| `rejected` | 已拒绝，余额已退回 |
| `processing` | 转账处理中 |
| `completed` | 转账完成 |
| `failed` | 转账失败，余额已退回 |

### 自动拆单逻辑

由于微信企业付款单笔限额200元，系统自动处理：

```typescript
// 示例：提现500元
// 自动拆分为：200 + 200 + 100 = 3笔转账
const amount = 500;
const maxSingle = 200;
const transferCount = Math.ceil(amount / maxSingle); // 3笔

transfers = [
  { no: 1, amount: 200 },
  { no: 2, amount: 200 },
  { no: 3, amount: 100 }
];
```

- 查看提现转账详情

## 支付系统

### 支付方式

#### 1. 支付宝支付
- **场景**: 用户充值、购买会员、购买课程
- **接口**: Alipay SDK
- **配置**: `.env` 文件中的 ALIPAY_* 配置
- **回调**: `/api/payment/alipay/notify`
- **跳转**: `/smartapi/payment/result`

#### 2. 微信支付
- **场景**: 用户充值、购买会员、购买课程
- **接口**: WeChat Pay SDK
- **配置**: `.env` 文件中的 WECHAT_* 配置
- **回调**: `/api/payment/wechat/notify`
- **跳转**: `/smartapi/payment/result`

#### 3. 微信企业付款（提现）
- **场景**: 用户提现到微信零钱
- **接口**: WeChat Transfer API
- **配置**: `.env` 文件中的 WECHAT_* 配置
- **限制**: 单笔最多200元，自动拆单

### 支付流程

```
用户发起支付
    ↓
选择支付方式（支付宝/微信）
    ↓
后端创建订单
    ↓
调用支付平台API
    ↓
返回支付链接/二维码
    ↓
用户完成支付
    ↓
支付平台回调通知
    ↓
验证签名
    ↓
更新订单状态
    ↓
更新用户余额/会员状态
    ↓
触发佣金计算
    ↓
完成
```

#### 2. 后端部署（backend）
```bash
1. 本地构建: npm run build
2. 创建部署包: tar -czf backend-deploy.tar.gz dist/ package.json
3. 上传到服务器: /www/smartapi/backend-deploy.tar.gz

5. 重启PM2服务: pm2 restart smartapi
6. 验证服务状态
```

```nginx
server {
    listen 80;
    server_name contentcube.cn;

        alias /www/smartapi/frontend/dist/;
        try_files $uri $uri/ /smartapi/index.html;

    # 管理后台
    location /admin/ {
        alias /www/smartapi/admin/dist/;
        try_files $uri $uri/ /admin/index.html;

        proxy_pass http://localhost:3000/api/;

        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

- `PUT /api/admin/users/:id/commission` - 设置用户佣金
- `GET /api/admin/withdrawals` - 获取提现申请列表
- `PUT /api/admin/withdrawals/:id/approve` - 批准提现
- `PUT /api/admin/withdrawals/:id/reject` - 拒绝提现
- `POST /api/admin/withdrawals/:id/transfer` - 执行转账

### 系统配置
- `GET /api/system-config` - 获取所有系统配置
- `PUT /api/system-config` - 更新系统配置（管理员）

### 支付相关
- `POST /api/payment/alipay/create` - 创建支付宝订单
- `POST /api/payment/wechat/create` - 创建微信订单
- `POST /api/payment/alipay/notify` - 支付宝回调
- `POST /api/payment/wechat/notify` - 微信回调

## 注意事项

### 安全相关
1. ⚠️ 所有敏感配置（密钥、证书）必须存放在 `.env` 文件，不能提交到Git
2. ⚠️ 数据库密码、JWT密钥需定期更换
3. ⚠️ 微信/支付宝证书文件权限设置为600（仅所有者可读写）
4. ⚠️ 管理后台必须添加IP白名单或强密码策略

### 数据备份
1. 数据库每天自动备份
2. `.env` 文件需要单独备份
3. 微信/支付宝证书文件需要备份

### 性能优化
1. SystemConfig 缓存TTL = 1分钟
2. 静态资源启用CDN加速
3. 数据库连接池配置合理
4. Nginx启用gzip压缩

### 监控告警
1. PM2进程监控
2. API接口响应时间监控
3. 支付回调失败告警
4. 提现异常告警
