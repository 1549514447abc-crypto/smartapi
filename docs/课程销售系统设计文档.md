# 课程销售系统 - 功能设计文档

## 📋 项目概述

这是一个**单课程销售系统**，主要特点：
- 只销售一门课程（可随时更换课程内容）
- 提供10节免费试听课程
- 支付后通过微信人工交付正式课程
- 正式课程不放在网站上，只放试听内容

---

## 🎯 核心需求分析

### 用户端需求
1. **课程介绍页** - 展示课程信息、讲师介绍、课程大纲、购买按钮
2. **试听页面** - 提供10节免费试听课程，用户可以直接在线观看
3. **支付流程** - 点击购买后生成订单，显示订单号和您的微信二维码
4. **订单查看** - 用户可以查看自己的购买记录

### 管理端需求
1. **课程信息管理** - 修改课程标题、描述、价格、讲师信息等
2. **试听课管理** - 上传/删除/编辑10节试听课（标题、视频、排序）
3. **订单管理** - 查看所有订单、标记已添加微信、备注信息

## 💾 数据库设计

### 表1：course_settings（课程设置表）
```sql
CREATE TABLE course_settings (
  id INT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(200) NOT NULL COMMENT '课程标题',
  subtitle VARCHAR(300) COMMENT '课程副标题',
  description TEXT COMMENT '课程描述',
  cover_image VARCHAR(500) COMMENT '课程封面图路径',
  original_price DECIMAL(10,2) COMMENT '原价',
  current_price DECIMAL(10,2) COMMENT '现价',

  instructor_name VARCHAR(100) COMMENT '讲师姓名',
  instructor_avatar VARCHAR(500) COMMENT '讲师头像路径',
  instructor_bio TEXT COMMENT '讲师简介',

  highlights JSON COMMENT '课程亮点（数组）',
  outline JSON COMMENT '课程大纲（章节数组）',

  wechat_qr_image VARCHAR(500) COMMENT '微信二维码图片路径',

  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 表2：course_lessons（试听课程表）
```sql
CREATE TABLE course_lessons (
  id INT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(200) NOT NULL COMMENT '课程标题',
  video_path VARCHAR(500) NOT NULL COMMENT '视频文件路径',
  duration INT COMMENT '视频时长（秒）',
  sort_order INT DEFAULT 0 COMMENT '排序序号',

  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  INDEX idx_sort (sort_order)
);
```

### 表3：course_orders（订单表）
```sql
CREATE TABLE course_orders (
  id INT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(50) UNIQUE NOT NULL COMMENT '订单号',
  user_id INT NOT NULL COMMENT '用户ID',

  course_title VARCHAR(200) COMMENT '课程标题',
  amount DECIMAL(10,2) NOT NULL COMMENT '支付金额',

  status ENUM('pending', 'completed', 'cancelled') DEFAULT 'pending' COMMENT '订单状态',
  admin_note TEXT COMMENT '管理员备注',

  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  INDEX idx_user (user_id),
  INDEX idx_order_no (order_no),
  INDEX idx_status (status)
);
```

## 🔧 技术实现方案

### 后端技术栈
- **框架：** Node.js + Express + TypeScript（复用现有架构）
- **文件上传：** Multer（保存到 `/uploads/course/` 目录）
- **视频播放：** 使用流式传输（支持断点续传）
- **认证：** JWT Token（复用现有用户系统）

### 前端技术栈
- **框架：** React + TypeScript（复用现有架构）
- **UI组件：** Ant Design
- **视频播放：** HTML5 `<video>` 标签
- **路由：** React Router

### 文件存储结构
```
/uploads/
  /course/
    /cover/         # 课程封面图
      - cover.jpg
    /instructor/    # 讲师头像
      - avatar.jpg
    /lessons/       # 试听课视频
      - lesson-1.mp4
      - lesson-2.mp4
      - ...
    /wechat/        # 微信二维码
      - qr-code.jpg
```

## 📡 API接口设计

### 用户端接口

#### 1. 获取课程信息
```
GET /api/course/info
Response: {
  title, subtitle, description, cover_image,
  original_price, current_price,
  instructor: { name, avatar, bio },
  highlights: [...],
  outline: [...]
}
```

#### 2. 获取试听课列表
```
GET /api/course/lessons
Response: {
  lessons: [
    { id, title, duration, sort_order },
    ...
  ]
}
```

#### 3. 获取试听课视频流
```
GET /api/course/lessons/:id/video
Response: 视频流（支持Range请求）
```

#### 4. 创建订单
```
POST /api/course/orders
Body: { user_id }
Response: {
  order_no, amount, wechat_qr_image
}
```

#### 5. 查看我的订单
```
GET /api/course/orders/my
Response: {
  orders: [
    { order_no, created_at, amount, status },
    ...
  ]
}
```

### 管理端接口

#### 6. 更新课程设置
```
PUT /api/admin/course/settings
Body: { title, subtitle, description, ... }
Response: { message: 'success' }
```

#### 7. 上传图片/视频
```
POST /api/admin/course/upload
Body: FormData (file)
Response: { file_path: '/uploads/course/...' }
```

#### 8. 管理试听课（CRUD）
```
POST   /api/admin/course/lessons        # 创建
GET    /api/admin/course/lessons        # 列表
PUT    /api/admin/course/lessons/:id    # 更新
DELETE /api/admin/course/lessons/:id    # 删除
PUT    /api/admin/course/lessons/:id/sort  # 调整排序
```

#### 9. 管理订单
```
GET /api/admin/course/orders            # 订单列表
PUT /api/admin/course/orders/:id/status # 更新状态
PUT /api/admin/course/orders/:id/note   # 添加备注
```

## ✅ 关键确认点

请您确认以下理解是否正确：

1. ✅ **只有一门课程** - 系统同时只展示一门课程，但您可以随时在后台替换课程内容
2. ✅ **10节试听课** - 用户可以免费在线观看10节试听课，视频存储在您的服务器上
3. ✅ **不自动支付** - 点击购买后只生成订单号和显示二维码，不对接支付宝/微信支付
4. ✅ **人工交付** - 用户添加微信后，您人工确认并通过微信发送完整课程（不通过网站）
5. ✅ **后台可替换** - 您可以随时在后台替换课程信息、试听视频、价格等
6. ✅ **本地存储** - 图片和视频直接存储在您的服务器上，不使用OSS

## 🤔 需要补充确认的问题

1. **用户系统** - 是否需要用户注册登录才能购买？还是不需要登录，只需填写姓名+手机号？
   - 方案A：必须登录（复用现有用户系统）
   - 方案B：不需要登录，创建订单时填写联系信息

2. **管理员权限** - 是否需要单独的管理员角色？还是直接用您现有的用户账号？
   - 方案A：现有用户表加 `is_admin` 字段
   - 方案B：单独创建管理员表

3. **课程大纲** - 介绍页面的课程大纲是手动编辑的文本，还是自动根据试听课生成？
   - 方案A：手动编辑（更灵活，可以写完整课程的章节）
   - 方案B：自动显示10节试听课列表

4. **视频格式** - 上传的视频需要做格式转换吗？还是直接存储原格式？
   - 方案A：直接存储（要求上传MP4格式）
   - 方案B：后端转换成统一格式（需要FFmpeg）
