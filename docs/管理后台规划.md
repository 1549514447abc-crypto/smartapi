### 1.2 现有数据模型

#### Users 用户表
```typescript
{
  id: number;                    // 用户ID
  username: string;              // 用户名
  email: string | null;          // 邮箱
  phone: string | null;          // 手机号
  nickname: string | null;       // 昵称
  avatar_url: string | null;     // 头像
  status: 'active' | 'suspended'; // 账户状态
  user_type: 'normal' | 'admin';  // 用户类型
  balance: number;               // 充值余额（不可提现）
  commission_balance: number;    // 返佣余额（可提现）
  points: number;                // 推广积分
  referral_level: number;        // 推广等级 1-3
  total_recharged: number;       // 累计充值
  total_consumed: number;        // 累计消费
  membership_type: 'none' | 'yearly' | 'course'; // 会员类型
  membership_expiry: Date | null; // 会员到期时间
  is_course_student: boolean;    // 是否课程学员
  created_at: Date;
  last_login_at: Date | null;
}
```

#### Commissions 佣金记录表
```typescript
{
  id: number;
  referrer_id: number;           // 推广人ID
  referee_id: number;            // 被推广人ID
  referral_id: number;           // 推广关系ID
  amount: number;                // 佣金金额
  commission_rate: number;       // 佣金比例
  source_amount: number;         // 来源金额
  source_type: 'recharge' | 'consume' | 'course' | 'membership'; // 来源类型
  status: 'pending' | 'settled' | 'cancelled'; // 状态
  settled_at: Date | null;
  created_at: Date;
}
```

#### CourseOrders 课程订单表
```typescript
{
  id: number;
  order_no: string;              // 订单号
  user_id: number;               // 用户ID
  course_title: string | null;   // 课程标题
  amount: number;                // 支付金额
  status: 'pending' | 'completed' | 'cancelled';
  admin_note: string | null;
  created_at: Date;
}
```

## 二、缺失功能分析

对比参考页面，以下功能需要新增：

### 2.1 财务报表模块 (高优先级)
**参考**: `管理端--财务报表.html`

需要展示的数据：
- **收入概览**：今日/本周/本月收入
- **充值统计**：充值金额趋势图、充值订单列表
- **消费统计**：消费金额趋势图、按功能分类统计
- **返佣统计**：返佣支出、待结算佣金
- **课程销售**：课程订单统计、销售额

数据来源：
- `users.total_recharged` - 累计充值
- `users.total_consumed` - 累计消费
- `commissions` - 返佣记录
- `course_orders` - 课程订单
- 需新增: `recharge_orders` 充值订单表 或 从现有数据聚合

### 2.2 用户财务模块 (高优先级)
**参考**: `管理端--用户财务.html`

需要展示的数据：
- **用户余额列表**：充值余额、返佣余额、积分
- **充值记录**：用户充值历史
- **消费记录**：用户消费历史
- **提现申请**：返佣提现审核
- **余额调整记录**：管理员调整历史

数据来源：
- `users` - 用户余额信息
- 需新增: `balance_logs` 余额变动日志表
- 需新增: `withdrawal_requests` 提现申请表

### 2.3 系统监控模块 (中优先级)
**参考**: `管理端--系统监控.html`

需要展示的数据：
- **服务状态**：API服务、数据库、缓存状态
- **API调用统计**：调用量趋势、错误率
- **性能指标**：响应时间、CPU/内存使用
- **在线用户**：当前活跃用户数

数据来源：
- `api_call_logs` - API调用日志
- 需新增: 系统健康检查接口

### 2.4 价格管理模块 (中优先级)
**参考**: `管理端--价格管理页面.html`

需要管理的价格：
- **会员价格**：年度会员 ¥299、课程学员 ¥799
- **功能定价**：视频提取、各API调用费用
- **优惠设置**：折扣规则、促销活动

数据来源：
- `system_configs` - 系统配置（已有）
- 需新增: `pricing_rules` 定价规则表（可选）

## 四、新增API接口规划

### 4.1 财务报表接口
```typescript
// 获取财务概览
GET /api/admin/finance/overview
Response: {
  today: { recharge: number, consume: number, commission: number },
  week: { recharge: number, consume: number, commission: number },
  month: { recharge: number, consume: number, commission: number }
}

// 获取收入趋势
GET /api/admin/finance/trend?type=recharge|consume&days=7|30
Response: {
  data: [{ date: string, amount: number }]
}

// 获取充值订单列表
GET /api/admin/finance/recharge-orders?page=1&pageSize=20
Response: {
  list: RechargeOrder[],
  total: number
}
```

### 4.2 用户财务接口
```typescript
// 获取余额变动日志
GET /api/admin/users/:id/balance-logs?page=1&pageSize=20
Response: {
  list: BalanceLog[],
  total: number
}

// 获取提现申请列表
GET /api/admin/withdrawals?status=pending|approved|rejected
Response: {
  list: WithdrawalRequest[],
  total: number
}

// 审核提现申请
PUT /api/admin/withdrawals/:id
Body: { status: 'approved' | 'rejected', remark: string }
```

### 4.3 系统监控接口
```typescript
// 获取系统状态
GET /api/admin/system/health
Response: {
  api: 'ok' | 'error',
  database: 'ok' | 'error',
  uptime: number,
  memory: { used: number, total: number }
}

// 获取API调用统计
GET /api/admin/system/api-stats?days=7
Response: {
  total: number,
  success: number,
  error: number,
  trend: [{ date: string, count: number, errorRate: number }]
}
```

## 五、新增数据表规划

### 5.1 balance_logs 余额变动日志
```sql
CREATE TABLE balance_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  type ENUM('recharge', 'consume', 'commission', 'withdrawal', 'admin_adjust'),
  balance_type ENUM('balance', 'commission_balance', 'points'),
  amount DECIMAL(10, 4) NOT NULL,  -- 变动金额（正负）
  balance_before DECIMAL(10, 4),   -- 变动前余额
  balance_after DECIMAL(10, 4),    -- 变动后余额
  source_type VARCHAR(50),         -- 来源类型
  source_id BIGINT,                -- 来源ID
  operator_id BIGINT,              -- 操作人ID（管理员调整时）
  remark TEXT,
  created_at DATETIME DEFAULT NOW()
);
```

### 5.2 withdrawal_requests 提现申请
```sql
CREATE TABLE withdrawal_requests (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  amount DECIMAL(10, 2) NOT NULL,
  status ENUM('pending', 'approved', 'rejected', 'paid') DEFAULT 'pending',
  payment_method VARCHAR(50),      -- 支付方式：alipay/wechat/bank
  payment_account VARCHAR(100),    -- 收款账号
  payment_name VARCHAR(50),        -- 收款人姓名
  admin_id BIGINT,                 -- 审核管理员
  admin_remark TEXT,
  paid_at DATETIME,
  created_at DATETIME DEFAULT NOW(),
  updated_at DATETIME DEFAULT NOW()
);
```
